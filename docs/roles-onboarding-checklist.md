# OctoAcme — Roles Onboarding Checklist

## Purpose
This document helps new contributors get up to speed quickly by providing a common first-two-weeks checklist and role-specific onboarding tasks. It complements the [Roles & Personas](octoacme-roles-and-personas.md) and [Role Interactions & RACI](role-interactions-and-raci.md) documents.

---

## Common — First 2 Weeks (All Roles)

- [ ] Read the [Project Management Overview](octoacme-project-management-overview.md)
- [ ] Read the [Roles & Personas](octoacme-roles-and-personas.md) to understand how all roles collaborate
- [ ] Review the [Role Interactions & RACI](role-interactions-and-raci.md) for your area of responsibility
- [ ] Set up access to the project board (e.g., GitHub Projects)
- [ ] Attend your first daily standup and introduce yourself
- [ ] Review the current sprint backlog and in-progress items
- [ ] Schedule 1:1 introduction meetings with PM and PdM
- [ ] Join relevant communication channels (Slack/Teams, mailing lists)
- [ ] Review the latest release notes and project retrospective notes

---

## Scrum Master

### Artifacts to Review
- [ ] Current sprint backlog and Definition of Ready / Definition of Done
- [ ] Previous retrospective action items and their status
- [ ] Team velocity chart and burndown history
- [ ] Impediment log (active and resolved)

### Meetings to Attend
- [ ] Shadow at least one full sprint cycle (planning → standup → review → retrospective) before facilitating
- [ ] 1:1 with PM to understand external commitments and escalation paths
- [ ] 1:1 with PdM to understand backlog health and upcoming priorities

### Initial Tasks
- [ ] Confirm ceremony schedule and update calendar invites if needed
- [ ] Introduce yourself and your role to the full team at the next standup
- [ ] Review the RACI matrix and identify any current process gaps
- [ ] Establish or review team working agreements

---

## UX Designer

### Artifacts to Review
- [ ] Product roadmap and current sprint goals
- [ ] Existing wireframes, prototypes, and design system or style guide
- [ ] User research findings and usability test reports
- [ ] Acceptance criteria for in-flight features

### Meetings to Attend
- [ ] Design review session (ongoing cadence)
- [ ] Sprint planning and sprint review/demo
- [ ] 1:1 with PdM to align on product vision and upcoming design needs

### Initial Tasks
- [ ] Gain access to the design tool (Figma, Sketch, etc.)
- [ ] Review current in-progress feature designs and flag any open questions
- [ ] Identify any user research gaps for the next sprint's features
- [ ] Add yourself as a reviewer on relevant PRs that touch the UI

---

## Technical Writer

### Artifacts to Review
- [ ] Existing docs in `docs/` and any API documentation
- [ ] Latest release notes and changelog
- [ ] Documentation contribution guidelines (if any)
- [ ] Current sprint deliverables to identify upcoming documentation needs

### Meetings to Attend
- [ ] Sprint planning (to identify documentation tasks for the sprint)
- [ ] Sprint review/demo to understand what has shipped
- [ ] 1:1 with Developers for any in-progress features requiring docs

### Initial Tasks
- [ ] Audit existing documentation for accuracy and flag outdated content
- [ ] Add documentation tasks to the backlog for any undocumented features
- [ ] Set up or verify the documentation publication pipeline
- [ ] Establish a process with PM for release note review and sign-off

---

## Business Analyst

### Artifacts to Review
- [ ] Product roadmap and high-level business goals
- [ ] Existing business requirements documents (BRDs) and user stories
- [ ] Requirements traceability matrix (if it exists)
- [ ] Previous workshop outputs and decision logs

### Meetings to Attend
- [ ] Backlog refinement / grooming sessions
- [ ] Requirements workshops with Stakeholder Representatives
- [ ] 1:1 with PdM to understand product strategy and upcoming features

### Initial Tasks
- [ ] Review the current backlog for stories that lack sufficient acceptance criteria
- [ ] Schedule introductory meetings with key Stakeholder Representatives
- [ ] Identify any requirements gaps for the current sprint or upcoming release
- [ ] Familiarise yourself with the escalation path for conflicting requirements (see [RACI](role-interactions-and-raci.md))

---

## Stakeholder Representative

### Artifacts to Review
- [ ] Project Charter / One-pager and success metrics
- [ ] Roadmap and current release plan
- [ ] Latest status report and risk register
- [ ] RACI matrix to understand your approval responsibilities

### Meetings to Attend
- [ ] Monthly stakeholder update meeting
- [ ] Sprint demo/review (at minimum, the end-of-sprint milestone demos)
- [ ] Kick-off meeting for any new project phase

### Initial Tasks
- [ ] Confirm your communication preferences with the PM (frequency, format)
- [ ] Review and confirm the project's stated business goals and success metrics
- [ ] Identify any business constraints or deadlines the team should be aware of
- [ ] Review pending sign-off items in the backlog and prioritise as needed
